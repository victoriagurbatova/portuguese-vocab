# Portuguese Vocab — MVP

Веб-приложение для изучения португальской лексики: карточки, режимы «показать ответ» и «ввод ответа», прогресс в localStorage.

## Запуск

```bash
npm install
npm run dev
```

Открой [http://localhost:3000](http://localhost:3000).

## Где положить CSV со словами

1. **Папка:** `data/` в корне проекта (рядом с `package.json`).
2. **Имя файла:** `vocabulary.csv`.

Полный путь: `portuguese-vocab/data/vocabulary.csv`.

Если файла нет или он пустой/битый, приложение подставит встроенный набор из нескольких слов (mock).

## Формат CSV

Первая строка — заголовки. Ожидаемые колонки (порядок может быть любой, учитываются имена):

| Колонка           | Описание                                      |
|-------------------|-----------------------------------------------|
| `id`              | Уникальный идентификатор (строка или число)   |
| `portuguese`      | Слово/фраза на португальском                  |
| `pronunciation_ru`| Транскрипция русскими буквами (по желанию)     |
| `russian`         | Перевод на русский                            |
| `category`        | Тематическая категория (например, «еда»)      |
| `examples`        | Пример использования (по желанию)              |
| `type`            | Тип (regular, irregular_verb, country и т.д.) |

Разделитель — запятая. Если в ячейке есть запятые или переносы строк, оберни её в кавычки.

Пример:

```csv
id,portuguese,pronunciation_ru,russian,category,examples,type
1,olá,ола,привет,приветствия,"Olá! Como vai?",regular
2,água,агуа,вода,напитки,"Uma água, por favor.",regular
```

## Как поменять путь к CSV

Файл читается на сервере в API-роуте. Путь задаётся в `src/app/api/vocabulary/route.ts`:

```ts
const CSV_PATH = path.join(process.cwd(), "data", "vocabulary.csv");
```

Измени `"data"` и/или `"vocabulary.csv"` под свою структуру (например, `path.join(process.cwd(), "content", "words.csv")`).

## Категории и типы

- **category** и **type** берутся из CSV и используются для фильтров в «Библиотеке» и для выбора набора слов в «Учить» (по категории или по типу).
- Списки категорий и типов строятся автоматически по данным из CSV.

## Mock-данные

Если `data/vocabulary.csv` отсутствует или после разбора не получается ни одной строки, API отдаёт встроенный массив из нескольких слов (`src/lib/mockData.ts`). Так приложение всегда можно запустить и проверить без своего файла.

```bash
npm run build
```
